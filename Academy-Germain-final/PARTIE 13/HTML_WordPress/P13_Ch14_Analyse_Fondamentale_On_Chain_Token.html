📖 Définition
L'Analyse on-chainOn-chain analysis — Analyse on-chain
Analyse des données directement sur la blockchain. Permet de voir les mouvements des whales, les flux vers les exchanges, l'adoption réelle. consiste à analyser directement les données de la blockchain pour comprendre les comportements des participants.
La blockchain est publique — chaque transaction est visible par tout le monde.
C'est l'équivalent de combiner le Form 4 (mouvements d'insiders), le 13F (mouvements institutionnels) et le bilan en temps réel.

Exchange InflowExchange Inflow — Flux entrant vers les exchanges
Envoi de cryptos vers un exchange — signal de vente imminente. Mouvement visible sur la blockchain avant toute transaction./Outflow — Le Signal le Plus Puissant

Quand une whale ou un institutionnel veut vendre ses Bitcoins, il doit d'abord les envoyer vers un exchange. Ces mouvements sont visibles sur la blockchain.

📊 Exemple concret
Mars 2020 (COVID crash) : avant la chute de -50% en une journée, l'inflow vers les exchanges avait explosé quelques jours avant.
Novembre 2022 (crash FTX) : les données on-chain montraient des sorties massives de FTX vers des wallets privés — signal d'alarme avant l'annonce officielle.
Fin 2020 / début 2021 (bull run) : outflow massif depuis les exchanges pendant des mois — les holders retiraient pour HODL = pas de vente = hausse.
Règle : Exchange OutflowExchange Outflow — Flux sortant des exchanges
Retrait de cryptos depuis un exchange vers des wallets privés — signal d'accumulation long terme. constant + prix stable = accumulation silencieuse = potentiellement haussier.
L'Activité GitHub — Le Code ne Ment Pas
📖 Définition
La plupart des projets crypto sont open source — leur code est public sur GitHub.
L'activité GitHub est l'un des meilleurs indicateurs de la santé réelle d'un projet.
Un projet avec un GitHub mort depuis 6 mois = équipe abandonnée ou projet fantôme.

Comment analyser le GitHub d'un projet crypto :

• Aller sur github.com → chercher le nom du projet
• Vérifier les 'Commits' récents : y a-t-il du code écrit dans les 30 derniers jours ?
• Vérifier le nombre de 'Contributors' : combien de développeurs travaillent activement ?
• Lire les 'Issues' ouvertes : des bugs connus non corrigés depuis longtemps ? Mauvais signe.
• Regarder les 'Stars' et 'Forks' : indicateurs de l'intérêt de la communauté développeurs

< 10 commits/moisDéveloppement quasi arrêté. Attention.Beaucoup d'altcoins en déclin

Un bon projet peut avoir peu d'activité GitHub si son code est 'fini' (cas rare). Mais vérifier les issues et pull requests.

📊 Exemple concret
Ethereum en 2023 : génère ~1-2 milliards$/an de revenus. Market Cap ~200 milliards$. P/F ≈ 100-200. Cher mais justifié par l'adoption.
Uniswap : génère des centaines de millions en frais. Market Cap ~5 milliards. P/F plus raisonnable.
Comparaison : un token avec 0$ de revenus mais 1 milliard$ de market cap → P/F = infini → 100% spéculatif.
Règle : pour investir long terme, chercher des projets avec Market Cap / Revenus < 50.
Comment Savoir si une Technologie est Réellement Utilisée

La question fondamentale avant d'investir dans une crypto : est-ce que des gens réels utilisent vraiment cette technologie dans la vraie vie ?

Question à poserComment y répondreSignal positifSignal négatif
Des entreprises réelles l'utilisent-elles ?Chercher des partenariats vérifiables sur le site + newsIBM + Stellar, Visa + SolanaPartenariats annoncés mais non confirmés
Le réseau est-il utilisé organiquement ?Transactions quotidiennes + active addresses sur DeFiLlama/GlassnodeCroissance organique sans incentivesVolume soutenu uniquement par des récompenses de staking
Les développeurs construisent-ils dessus ?GitHub + nombre de dApps déployéesÉcosystème dApps croissantPeu de projets construits sur le réseau
Les frais de transaction sont-ils payés ?Revenue sur Token TerminalRevenus croissants = utilisation réelleRevenus proches de 0 = personne n'utilise vraiment
Y a-t-il une vraie concurrence à une solution existante ?Comparer avec alternatives centraliséesRésout un vrai problème mieux que les solutions actuellesCopie d'une solution déjà existante sans amélioration réelle
📊 Exemple concret
Chainlink : fournit des données réelles (prix, météo...) aux smart contracts. Utilisé par des centaines de protocoles DeFi. Revenus réels. Technologie indispensable à l'écosystème.
Ethereum : la blockchain la plus utilisée par les développeurs. Des milliers de dApps. Des milliards de frais générés.
Exemple négatif : XRP (Ripple) — présenté comme 'la solution de paiement bancaire mondial'. Adoptée par quelques banques pour des tests. Mais SWIFT (la vraie solution bancaire) continue d'être utilisé dans 99% des cas.
Règle d'or : si tu ne peux pas nommer 3 entreprises ou projets réels qui utilisent activement cette technologie — c'est peut-être une solution en recherche de problème.
